SoftTimer 

SoftTimer implementation suitable for resource constrained environments like embedded systems

Status

  1. Work in Progress

Code Structure

  1. Source files are present in Src
  2. Interface files are present in inc
  3. PLease refer to the generated HTML documentation in the Doc/html subfolder for implementation details
  4. The function entry point is Main.c

Usage

  1. softTimer_Register with timeout and call back to be invoked. Captured the returned variable as the ID that can be used to Disarm or Deregister the Timer
  2. Update MAX_TIMERS with the maximum timers needed. The memory is statically allocated though global table gSoftTimerTable
  1. softTimer_PeriodicTask should be called by a hardware timer at the required resolution. All timers will be scheduled in multiples of this periodicity. Update Macro TIMER_RESOLUTION_MS with this value
  2. softTimer_StartAll and softTimer_StopAll can be used to start or stop the timers
